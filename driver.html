Driver Functions¶

On this page, the functions pertaining to the driver script are explained. Each function in the driver performs a specific function in the knowledge base pipeline.

laura_library.all(keywords, max_results, working_dir, num_topics, JAR_loc, vis_type, spert_config, output_filename, vis_dir)¶

Runs all modules from start (scraping) to finish (knowledge graph)

Arguments: keywords -- string list of keywords used to scrape for data max_results -- integer amount of results desired from scraping working_dir -- working directory of all actions num_topics -- number of topics to cluster for in analysis JAR_loc -- path to the JAR file for CERMINE vis_type -- string, one of three choices: wordcloud, pyldavis, scatter

Returns: nothing, side effects in working directory passed in

laura_library.analyze(working_dir, num_topics)¶

Analyzes Gensim dictionary and corpus and saves as LDA model.

Arguments: working_dir -- working directory path num_topics -- number of topics LDA is given

Returns: Nothing. Side effect is LDA model saved to working directory.

laura_library.convert(working_dir, JAR_location)¶

Uses CERMINE to convert PDFs to CERMXML format.

Arguments: working_dir -- working directory path JAR_location -- path to CERMINE JAR executable

Returns: Nothing. Side effects in working_dir

laura_library.create(working_dir)¶

Creates Gensim dictionary and corpus based on JSON files in working directory.

Arguments: working_dir -- working directory path

Returns: Nothing. Side effect is saved gensim dict and corpus to working_dir.

laura_library.parse(working_dir)¶

Parses CERMXML files to dictionary (JSON) format.

Arguments: working_dir -- working directory path

Returns: Nothing. Side effect of JSON files in working_dir.

laura_library.scrape(keywords, max_results, working_dir)¶

Scrapes arXiv for dataset.

Arguments: keywords -- keyword queries to scrape dataset on max_results -- maximum number of results wanted (upper bound) working_dir -- working directory path

Returns: Nothing. Data is collected to working_dir

laura_library.visualize(vis_type, working_dir)¶

Visualizes the LDA model in one of three ways: wordcloud, scatter plot, or using PyLDAVis.

Arguments: vis_type -- "wordcloud", "scatter", or "pyldavis" working_dir -- working directory path, LDA model must be saved here.

Returns: Nothing. Side effect is visualization saved to working directory.
